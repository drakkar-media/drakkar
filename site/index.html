Self-hosted · Usenet-native

Watch it before it finishes downloading.

Drakkar is a self-hosted Usenet media manager for Plex and Jellyfin. It finds releases, streams them straight from Usenet through a virtual filesystem, and hands them off the moment they're requested — no full download sitting between a request and playback.

$ cd drakkar && cp data/settings.example.json data/settings.json
$ docker compose up -d
# → http://localhost:8080

Capabilities

Everything a downloader does, minus the waiting

Drakkar replaces the download step entirely — content streams the moment it's needed, and the library stays consistent on its own.

Stream-first virtual filesystem

A FUSE + WebDAV layer serves media instantly. Articles are fetched from Usenet on read and cached to disk — playback starts before anything is fully downloaded.

Multi-provider fallback

A pooled, connection-aware NNTP client retries across every configured provider when an article is missing or throttled — one slow stream never starves the rest of the pool.

Smart season-pack fulfillment

Finds a season pack once and fulfills every requested episode from it — instead of running a separate search and download for each one.

Quality profiles & custom formats

Configurable resolution and codec preferences with cutoff support, plus regex-based custom format scoring layered on top of every search result.

Self-healing library

Periodic health checks catch missing symlinks, stale releases, and unreadable articles, then repair and republish automatically — and tell Plex or Jellyfin the moment they do.

Subtitle automation

Automatic subtitle search and download via OpenSubtitles and SubDL, republished alongside the media whenever a file is repaired or replaced.

Requests that just work

Overseerr or Jellyseerr requests flow straight into Drakkar's own queue. Or run it as a drop-in, SABnzbd-compatible download client for Radarr and Sonarr.

Notifications

Discord webhooks and generic HTTP webhooks fire on grab, import, and failure events, so you know what's happening without watching the logs.

Pipeline

From request to playback

Every step after the request is Drakkar's own pipeline — no intermediate download client, no waiting room.

01

Request

Overseerr, Jellyseerr, or Radarr/Sonarr ask for a title.

02

Search & rank

NZBHydra2 indexers are queried and results scored by quality profile.

03

Index

The NZB is fetched and indexed — no article bodies downloaded yet.

04

Publish

A virtual file appears instantly, symlinked into your library.

05

Stream

Plex or Jellyfin plays it — articles fetch on read, in real time.

Quick start

Running in about five minutes

All you need is Docker, a Usenet provider, and an indexer.

  • Docker & Docker Compose
  • A Usenet provider with SSL (NNTP port 563)
  • NZBHydra2 for indexing
  • Overseerr/Jellyseerr, or Radarr/Sonarr
$ git clone https://github.com/drakkar-media/drakkar
$ cd drakkar
$ cp data/settings.example.json data/settings.json
$ chmod 600 data/settings.json
# edit settings.json — add Usenet, NZBHydra2, Overseerr
$ docker compose up -d
# web UI → http://localhost:8080
